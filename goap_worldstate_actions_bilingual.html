§0概览/ Overview

问题： 原始 9 类粗粒度行为枚举（MOVE / STAY / STUDY_WORK / EAT_BUY / SOCIAL / USE_SPACE / GOVERNANCE / EMERGENCY / IDLE）粒度过粗，无法用作 GOAP 的搜索单元；同时多智能体架构里已固定双层 FSM——身体 move / interact / find / idle、对话 talk / mutter / phone_call。本文以这 7 个原子动作为搜索单元，重建一套极简 worldState + GOAP 前置/效果/代价，并用复合行为示范 GOAP 搜索如何把旧 9 类完整覆盖。 Problem: The original 9-bucket action enum (MOVE / STAY / STUDY_WORK / EAT_BUY / SOCIAL / USE_SPACE / GOVERNANCE / EMERGENCY / IDLE) is too coarse to serve as GOAP search atoms. The multi-agent stack already fixes a dual-layer FSM — Body move / interact / find / idle, Dialog talk / mutter / phone_call. We rebuild a minimal worldState + GOAP precond/effect/cost on these 7 atoms, and demonstrate via composite behaviors how the old 9 enums are fully recovered by GOAP search.

设计原则（4 点）

  • 分层解耦：场景树只放共享 Place 事实；agent 只放私有 11 字段；二者通过 location 路径耦合。Layer split: scene tree carries shared Place facts only; agents hold 11 private fields; coupled via a dotted location path.
  • 桶位优先：所有强度量（hunger/energy/focus/social_need/occupancy）用 0..3 桶位，money 用 0..2，避免浮点状态空间爆炸。Buckets over scalars: all intensities use 0..3 (money 0..2) integer buckets; no floats in state space.
  • 能枚举不字符串：mood / holding / role / noise 全部封闭枚举；新增需改 schema。Enums over free text: mood / holding / role / noise are closed enums; extension requires schema change.
  • 3 层定深：campus → district → place，物件粒度全部下沉到 agent.holding，杜绝"为剧情临时挂字段"。Fixed depth 3: campus → district → place; sub-place objects collapse into agent.holding; no ad-hoc fields per scenario.

§1WorldState 场景树/ Scene Tree

根节点 = campus；下分 6 个 district（教学/生活/运动/办公/边界/医疗），共 25 个 place。每个 place 仅挂 5–8 个核心字段；district 节点不挂字段，仅作语义分组与"区域级广播"挂点。Root = campus; 6 districts (teaching/life/sports/admin/edge/medical) host 25 places. Each place carries 5–8 core fields; districts hold no fields and serve only as grouping nodes and broadcast pivots.

  • agent.target_in_range == false

  • agent.location := path_to(target)

5.2 interact · 交互（通用占位）

交互interact(target) · body channel · affordance-drivenbodycost 1..4dur 1..3
最通用的占位动作：吃饭/上机/上课/休息全部走它。effects 由 target 的 affordance 字段（has_food / has_seat & quiet / has_compute / ddl_pressure）条件式决定，不在原子层硬编码。The universal placeholder. Eating / coding / lecturing / resting all route through this verb. Effects are conditional on target affordance fields (has_food / has_seat & quiet / has_compute / ddl_pressure) — never hard-coded per scenario at the atomic layer.
preconditions
  • agent.target_in_range == true (co-located with target)
  • target.is_open == true
  • target.occupancy < 3
  • if target.has_food: agent.money > 0
  • if target.has_compute: agent.holding ∈ {none, homework, tool}
  • agent.energy > 0 (除非 affordance == rest)
effects (条件式)
  • 通用：target.occupancy += 1; agent.last_action := "interact"
  • if target.has_food: hunger −= 2; money −= 1; holding := meal_done; mood ← calm
  • if target.has_seat && noise=="quiet": focus += 1
  • if target.has_compute: focus += 1; energy −= 1; holding := homework
  • if target.has_ddl_pressure: focus += 2; mood := anxious
  • if target.affordance == "sport": energy −= 2; mood := excited; hunger += 1
  • 所有数值 clamp 0..3
cost
1 + duration_bucket(0..2) + paid_term(0..1) → [1..4]
duration
1 (买咖啡) / 2 (吃饭/运动) / 3 (自习/写代码)
typical targets
Place（食堂/教室/机房/操场） · Object（自动售货机/床/外卖柜）
failure modes
occupancy 已满 / money 不足 / target.is_open 中途关闭 / holding 冲突（meal 想写代码） / affordance 与 role 不匹配

5.3 find · 寻找

寻找find(query) · body channel · cognitivebodycost 1..3dur 1..2
"位置未知"显式建模：将 knows_target_location 从 false 翻为 true。不要让 planner 把"不知道在哪"塞进 move 失败重试循环。Explicitly models "I don't know where it is": flips knows_target_location from false to true. Prevents planner from masking ignorance behind move-fail loops.

  • if 通过询问 NPC 完成: social_need −= 1
  • agent.last_action := "find"
  • 失败回退：自动发射子目标 interlocutor_available → planner 插入 talk(staff) 或 phone_call(friend)
cost
1 + uncertainty_bucket(0..2) → [1..3]
duration
1（问路成功）/ 2（无指示牌反复搜索）
typical targets
Place（陌生教室） · Agent（找某人） · Object（失物）
failure modes
周围无可询问者 + 无指示牌 / 反复 find 仍失败 → 抛回 planner / energy 耗尽中断

5.4 idle · 闲置

闲置idle · body channel · explicit restbodycost 1..3dur 1..2
显式休息动作（不是空状态）：energy 恢复，但 focus 涣散、social_need 累积。让 planner 主动选择"歇一会"，而不是陷入饥饿/疲劳死循环。An explicit rest action (not a null state): energy regenerates, focus drifts, social_need accumulates. Lets the planner actively choose a break instead of looping into hunger/fatigue.

  • agent.energy += 1 (clamp 0..3)
  • agent.social_need += 1 (轻微累积)
  • agent.focus −= 1 (注意力涣散)
  • if mood == tired: mood := calm
  • agent.last_action := "idle"
cost
1 + outdoor_bad_weather_penalty(0..2) → [1..3]
duration
1（发呆）/ 2（久坐）
typical targets
self / 可坐 Place
failure modes

5.7 phone_call · 电话 NEW

电话phone_call(target_agent) · dialog channel · remote · probabilisticdialogcost 2..4dur 1..2prob fail
远程双向通话。无距离限制（双方可在任意 place 甚至校外），但有概率打不通（被占线 / 静音 / 信号差），失败时 mood 轻微变 down，knows_target_location 不变。与 move 可并发——这是它存在的核心价值。Remote two-party call. No distance constraint (both ends may be at any place, even off-campus), but probabilistic connect failure (busy / muted / weak signal). On failure, caller's mood drifts toward down. Concurrent with move — this is its core value over talk.
preconditions
  • agent.in_dialog == null
  • agent.holding ∈ {none, phone, ...} (可空闲一只手 / one hand free)
  • target 是 Agent（任意 location，不要求 target_in_range / no co-location required）
  • target.role != special_no_phone (软约束)

为何 phone_call 与 talk 必须分开： (1) talk 强制要求 target_in_range（同位置），是 GOAP planner 决定"是否要先 move"的关键；如把电话归入 talk，planner 会错误地为"打电话"也插入 move(target)。(2) phone_call 的概率失败是真随机性，进入 worldState 的方式是更新 retry_count 与 mood，不是 plan-time 失败 — 这要求把它显式建模为带概率分支的动作。(3) 它能与 move 并发，这是 talk 严格禁止的；并发槽位规则不同。(1) talk mandates target_in_range (co-location), which the planner uses to decide whether to insert a move. Folding calls into talk would force unnecessary move(target). (2) phone_call's probabilistic failure is real stochasticity expressed via retry_count & mood updates — a runtime fact, not a plan-time failure. (3) It is concurrent with move, which talk strictly forbids; the concurrency slots differ.

§9GOAP 搜索条件/ Solvability & Optimality

定义。 给定初始世界状态 s₀、目标谓词 G、原子动作集 A。一条复合行为视为 GOAP 可解，当且仅当存在动作序列 a₁, …, aₙ ∈ A，同时满足以下三条：
(a) 前置可达：对任意 i，aᵢ.preconditions 在状态 sᵢ₋₁ = apply(s₀, a₁ … aᵢ₋₁) 下为真；
(b) 末态成目标：goal predicate G 在 sₙ 上求值为真；
(c) 代价最小：Σᵢ cost(aᵢ) 在所有满足 (a)(b) 的序列中最小，使用 A* 启发式 h(s) = |{p ∈ G : s ⊭ p}|（未满足子目标计数；admissible，因每条至少需 1 个动作消解）。 Definition. Given initial world state s₀, goal predicate G, and atomic action set A. A composite behavior is GOAP-solvable iff there exists an action sequence a₁, …, aₙ ∈ A satisfying:
(a) Precond reachability: for each i, aᵢ.preconditions hold in state sᵢ₋₁ = apply(s₀, a₁ … aᵢ₋₁);
(b) Goal in terminal state: G evaluates true at sₙ;
(c) Cost-minimal: Σᵢ cost(aᵢ) is minimum among all sequences satisfying (a)(b), under A* with heuristic h(s) = |{p ∈ G : s ⊭ p}| — count of unmet sub-goals; admissible since each unmet predicate needs at least one action to resolve.

9.1 复合行为存在性 ⇔ 子目标可分解

每条 §8 卡里的 chain 都是 (a)(b) 的构造性证明——给出动作序列即给出存在证明；总 cost 是上界，A* 在原子集合上闭合搜索得到下界，两者相等即最优。The chain in each §8 card is a constructive proof of (a)(b) — providing a sequence is providing existence; its total cost is an upper bound, and A* over the atom set yields the lower bound. Equality = optimality.

9.2 概率动作 phone_call 的处理

plan-time：把 phone_call 视作期望 cost 的确定性动作，cost = 2 + (1 − connect_p) × retry_cost ≈ 2 + 0.4 × 1 ≈ 2.4，舍入为 2。这样 A* 不需展开概率分支。
run-time：失败时仅修改 retry_count 与 mood，触发 L1 即时重排（见 §6 表）；retry≥2 时 planner 替换为 find / mutter。Plan-time: treat phone_call as a deterministic action with expected cost 2 + (1 − connect_p) × retry_cost ≈ 2.4, rounded to 2. A* never branches on probabilities.
Run-time: on failure, only retry_count and mood mutate; triggers L1 immediate replan; after retry ≥ 2 the planner substitutes find or mutter.

§13反膨胀策略/ Why this stays small

四道闸门 ⇒ 状态空间复杂度 O(places × 8 + agents × 11) 加性增长。
(1) 场景树只放共享事实，agent 只放私有状态；(2) 树深固定 3 层，物件粒度全部下沉到 holding；(3) 桶位 / 枚举 / 布尔代替自由量；(4) 派生字段（如 knows_target_location）每 tick 重算，不进 state hash。新增地点不动 agent，新增 agent 不动场景，"为剧情临时挂字段"被结构上拒绝。 Four gates ⇒ state space grows additively as O(places × 8 + agents × 11).
(1) scene tree carries shared facts, agents only private ones; (2) depth capped at 3, sub-place objects collapse into holding; (3) buckets / enums / bools replace free scalars and strings; (4) derived sensor fields (e.g. knows_target_location) are re-computed each tick and excluded from state hash. Adding places doesn't touch agents, adding agents doesn't touch scenes, ad-hoc per-scenario fields are structurally rejected.

13.2 拒绝清单

  • ❌ 把每件家具/小物挂到场景树（"床/桌子/咖啡机"）→ 用 holding 表达。❌ Hanging every piece of furniture on the scene tree ("bed/desk/coffee-machine") — use holding instead.
  • ❌ 给 mood 加 10+ 档（"轻微焦虑/中度焦虑/严重焦虑/濒临崩溃…"）→ 5 档枚举 + 派生 trigger。❌ Adding 10+ mood levels ("mild/medium/severe anxious…") — keep 5-tag enum + derived triggers.
  • ❌ 为每个 NPC 加私有 history 字段 → memory 走外部存储，不进 state hash。❌ Adding per-NPC history fields — memory goes to external storage, not state hash.
  • ❌ 在原子层硬编码"吃饭/学习/写代码"等场景动词 → 全部走 interact + 条件式 effects。❌ Hard-coding scene verbs ("eat/study/code") at atomic layer — all route through interact + conditional effects.
  • ❌ 把概率分支展开到 A* 搜索图 → 用期望 cost 折叠（§9.2）。❌ Expanding probabilistic branches into the A* graph — fold via expected cost (§9.2).

13.3 一句话总结

把 "做什么"（7 个原子动作 + 11 条复合行为）与 "在哪 / 谁 / 现在如何"（25 places × 8 字段 + 11 agent 字段）分开建模——这是这套设计能同时支持 60+ NPC 且不爆炸的根本原因。Decouple "what to do" (7 atoms + 11 composites) from "where / who / how" (25 places × 8 fields + 11 agent fields). That decoupling is why this design scales to 60+ NPCs without blowing up.
